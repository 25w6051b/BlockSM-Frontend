コード作成中に発生した疑問点や注意事項をまとめる

・blocklyでは文字列でコードを返し，最後にすべてのコードを実行する。(evel(code))
　${}を用いることで変数の中身にアクセスできる

・単純な文字列同士の比較はできない　理由は文字列コードで返す時に""を付け加えて返されるから？
　コンソール上では""がなく表示されているのでかなりの問題点
　console.logだけで表示内容を確認するのではなくJSON.stringify()を用いよう
  配列に関しても文字列で返すので，後で元の配列に戻そう
  
・条件式でfalseと"false"が使い分けしている理由
　前項で述べた通りだが，blocklyの関数を用いて何かしらの値を抜き出す時は""が強制的に付けられる
　よって抜き出した値をそのまま使いた場合は""で比較する
　自分で定義している時はtrue,falseで判定する(正直分かりづらいのでfalse以外で判定してもいいかも？可読性下がるけど)

・argsのinput_valueとinput_statementの違い
　-input_valueはブロックにブロックを囲う形で表現される
　 抜き出す時はタプル形式で抜き出すのが必須,codeだけでなく順序も設定しよう
　 例. return [code, javascript.javascriptGenerator.ORDER_ATOMIC];
  -input_statementはブロックに出口のある穴を作る形で表現される
   抜き出す時は単純なcodeだけ返せばいい
   例. return code

・ブロックの接続制限について
　ブロック同士で接続を制限することができる
　ブロック同士で同じ文字列を設定してあげればいい(※注意. 接続相手のtypeを設定すればいいわけではない)
　- ブロック同士で親子関係があるとき
　　親ブロックでは"check"で設定，子ブロックではoutputで設定可能
　- ブロック同士の階層が同じとき
　　previousStatementで前ブロック，nextStatementで後ブロックを設定可能

・ローカルストレージについて
　ローカルストレージはブラウザのドメインごとに保存される
　もしwebサイトのドメインをlocalhostにする場合，個人のPC内にデータが残り続ける可能性あり
　削除を呼びかけるか，簡単に削除できるUIは必須である
　ローカルストレージはキーが一意に決まっている
